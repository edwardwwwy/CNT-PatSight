# CNT-LitSight v1 项目边界

## 定位

CNT-LitSight v1 服务 CNT 研发组，用于在实验或专利实施例的 `run` 层级比较催化剂体系、CVD 工艺条件和 CNT 结果。第一版不追求构建 CNT 百科。

机器可读范围配置见 `config/project_scope.yaml`。

## 建议优先关注

以下内容用于安排第一批采集顺序，不是硬性纳入/排除边界：

- 路线：CVD / CCVD / 催化分解制备 CNT，以甲烷为首要碳源；天然气和其他 CVD 碳源可作为有价值对照。
- 来源：论文和专利。
- 催化剂：活性金属、载体、助剂、金属比例、前驱体和制备/活化条件。
- 工艺：反应器、温度、时间、压力、碳源、气氛、各气体流量和比例。
- 产品：SWCNT/MWCNT 等类型、产率口径、纯度、直径、长度、残留金属和表征证据。
- 后处理：酸洗、氧化、纯化及其条件。

## v1 要回答的问题

1. 哪些催化剂组合效果好？
2. 适合什么温度区间？
3. 对应什么碳源和气氛？
4. 产物是 SWCNT 还是 MWCNT？
5. 产率、纯度、直径、长度、残留金属如何？
6. 是否有酸洗、氧化等后处理？

## 可选补充

第一版不强求完整成本模型、穷举式表面表征、完整孔径分布、SWCNT 手性、金属/半导体比例或长期工业运行数据。来源若明确报告这些内容，可以保留为补充字段，但不因缺失而否定一个有研发价值的 run。

## 灵活性原则

- 建议字段不是硬性必填清单。
- 非甲烷碳源、非主流 CNT 类型、失败条件、负结果和不完整观察都可保留。
- 超出当前建议重点的来源仍可保存为候选、背景或对比资料，后续可重新提升优先级。
- 新出现且有研发价值的信息可扩展字段，或先进入 `notes` / `summary`，不为迁就现有 schema 而丢失。

## 证据边界

- 一篇来源可拆为多个 `run_id`；催化剂、温度、气体程序、时间或结果变化通常需要拆分。
- 专利实施例可形成 run；权利要求、背景和宽泛范围不能当作已完成实验。
- CNT 类型要区分作者声称与证据确认；SWCNT 证据不足时使用 `uncertain` 或 `unknown`。
- 产率必须保留原始定义和单位，不直接比较不同口径。
- 缺失值保持缺失，不猜测。
