# B 类可用全文重做协议 v1

## 范围

本轮只处理本地元数据中 `priority_tier=B` 且 `parse_source_status.parse_status=success` 的来源。2026-07-20 冻结时共有 139 篇。未取得全文的来源不进入本轮实验事实重建，也不得用摘要、旧八表或推断生成占位 run。

旧 B 类八表、旧 batch builder 和既有候选结果只可在最终差异审计阶段作为“待解释差异”，不得作为新值的证据来源。

## 来源优先级

1. 原始 PDF 页面、表格、图和补充材料；
2. 完整 scholarly HTML；
3. 与原始文件内容哈希绑定的 parsed section/span，用于检索和证据回填；
4. 摘要只能支持摘要中明确出现的事实，不能替代方法或逐 run 结果。

每篇 PDF 的关键实验表、相图、流程图和结果图必须渲染后视觉核查。文本抽取不能单独证明图表值或版面归属。

## Run 边界

一个 run 必须能指向一个可识别的催化剂/基底、工艺程序和对应结果。以下变化通常要求拆 run：催化剂配方、温度、压力、碳源、关键流量、反应时间、活化/再生状态、反应器尺度、后处理或独立结果系列。

论文只给系列范围而未给逐点结果时，保留 series-level 记录，不把范围内任意点实体化为精确 reported run。图上读取的点标为 `figure_digitized` 并保留近似性。

同一名义条件在不同表、系列或重复实验中出现不同值时，不静默去重。保留 series/replicate 归属和冲突问题记录。

## 禁止默认值

下列值只有原文明示时才能写入：

- `atmospheric`、`101.325 kPa` 或任何压力；
- `lab_batch`、bench、pilot 等规模；
- `fresh catalyst`；
- `0`、无产物或完全转化；
- CNT 类型、阵列、直径、长度、纯度或产率；
- 代表性 TEM/Raman/TGA 结果向全部 run 的继承。

未报告即留空或写 schema 允许的 `not_reported`，不得用常识补齐。

## 单位与值状态

原始单位和表达必须保留。规范化值通过 `evidence_index.value_status` 区分：

- `reported`：原文直接报告；
- `normalized`：只做同一物理量单位换算；
- `calculated`：按显式公式计算；
- `figure_digitized`：从图中读取；
- `inferred`：必要且可审计的推断，默认不进入关键筛选字段；
- `review_assessment`：成本、风险或产业化审阅判断。

带 `~`、about、approximately、范围或有效数字限制的值在换算后仍保留近似标志。不得把 K 字符串写入 `_C` 数值字段，也不得在不同产率定义之间换算。

## 证据

每个关键非空科学字段必须由 evidence 行覆盖，且定位到实际支持它的页、表、图或段落。证据不能只覆盖“这篇论文研究 CNT”，却被用于支撑具体尺寸、产率或样品状态。

证据必须记录：来源 ID、run ID、目标表、目标记录、目标字段、值状态、section、locator、原文片段和置信度。摘要/结论证据不得替代详细结果证据。

## 完整性与失败实验

逐篇核对论文表格、图例和实验描述中的 campaign 数量。失败、低产率或阴性实验可以成为正式 run，但成功产物字段必须为空，并明确记录失败结果。没有可重建 CNT 实验的来源标为 `non_extractable`，不生成虚假事实行。

## 审阅与晋级

重建初稿统一为 `needs_review/pending_review`。单篇晋级前必须完成：

1. 来源身份和本地文件哈希确认；
2. run 边界与实验成员数对账；
3. 关键字段逐项证据覆盖；
4. 单位、近似性、产率语义和值状态检查；
5. 主外键及 schema 校验；
6. 所有 high/critical 问题关闭；
7. PDF 关键图表视觉复核；
8. `validate_tables.py` 与 `validate_b_class_redo.py` 均为零错误。

最终合并前，由主审阅者对每个分片做风险分层复抽检，并对旧成果差异进行解释，但不因旧成果存在而覆盖新提取值。
